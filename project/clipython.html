1. Hybrid Encryption Core

RSA protects the exchange while AES handles the payload. The workflow stays explicit from key generation to decryption.

  • RSA-2048 handles secure key exchange before any payload work starts.
  • AES-256-CBC keeps bulk encryption fast and practical for real files.
  • Each step is staged so the security flow stays deterministic and easy to reason about.
🗂️

2. File & Message Workflow

Text and binary files use the same entry points, so the CLI stays compact even when switching targets.

  • Encrypt and decrypt short messages without changing the overall workflow.
  • Handle binary files through the same interface for a consistent user path.
  • Keep validation and error handling visible instead of hiding failures behind abstractions.
🔑

3. Key Management & Keystore

Generate, export, import, and organize key pairs for repeatable secure sessions.

  • Create RSA key pairs from the CLI without leaving the workflow.
  • Export and import keys to support reusable sessions and multiple environments.
  • Keep a structured keystore so future automation stays predictable.
💻

4. Terminal UX & Feedback

Menus, progress cues, and color-coded output make the tool readable even in long sessions.

  • Use compact menus so the CLI stays fast to navigate.
  • Surface progress and state clearly during longer operations.
  • Keep errors readable so debugging stays practical instead of opaque.
